#_container_#
there are three types of container

1. Sequence container :- (one by one)
    (i) vector (dynamic array) : that stores memory in continuous memory locations
    (ii) list (doublly linked-list) : that stores memory in doublly linked list
    (iii) deque (double handed queue) : where elements can be inserted and removed from both ends — the front and the back
    (iv) array (fixed-size) : it is a collection of elements of the same data type stored in continuous memory locations 
    (v) forward_list (singly linked-list) : that stores elements in a singly linked list format
[https://github.com/RutviParakhiya2007/CGxSU_Semester_1/blob/main/cpp_language(sem_02)/22.stl_containers.md#11-sequence-containers]    

2. Assosiative container :- (ordered / sorted)
    (i) set : ordered but unique element
    (ii) multiset : ordered but duplicate element
    (iii) map : unique (key) ordered 1. key value pair, 2. ordered (distinct element)
    (iv) multimap : duplicate key 1. count (distinct element), 2. erased
[https://github.com/RutviParakhiya2007/CGxSU_Semester_1/blob/main/cpp_language(sem_02)/22.stl_containers.md#12-associative-containers]

3. Unordered container :- (no order maintain)    
    (i) unordered set : unordered but unqiue element
    (ii) unordered multiset : unordered but duplicate element
    (iii) unordered map : unique (key) unordered 1. key value pair, 2. unordered (distinct element)
    (iv) unordered multimap : duplicate key 1. count (distinct element), 2. erased
[https://github.com/RutviParakhiya2007/CGxSU_Semester_1/blob/main/cpp_language(sem_02)/22.stl_containers.md#13-unordered-containers-in-c-stl]
